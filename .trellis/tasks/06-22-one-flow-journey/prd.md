# 一波流·周期制改造（子任务二 · 后做）

> 父任务：`06-22-offerpilot-reshape`。改的是流程外壳，与"匹配修复"正交。

## Goal

把当前"以对话为中心、可反复新建多会话"的信息架构，改成**"一个学习周期一波流"**：前期开场对话敲定方向 → 显式开始 → 进入旅程执行期且不回开场。让产品的心智从"聊天软件"回到"一段有始有终的求职冲刺"。

## 核心模型（已与用户逐项敲定，2026-06-22）

一个账号同一时间只跑**一个学习周期**。周期分两段：

```
【开场·分析期】对话编排①（沿用现有：web_search / analyze_match / generate_plan + 两步闸门）
   交简历+JD → 多轮聊 → 反问偏好 → 出计划      （可反复改，未锁）
        │
        ▼  generate_plan 出计划
【计划·待确认】只展示分周计划，journey = draft 态，尚未物化     （还能回去改）
        │
        ▼  ★ 用户显式点【开始这个学习周期】
   ═══ 此刻才：物化 Task + journey 转 active + 开场对话归档 ═══
        │
        ▼
【旅程·执行期】对话编排②（新：执行陪伴教练）
   · 旁边保留：任务看板 / 打卡 / 进度 / 再规划
   · 开场对话：折叠成只读归档（挂本周期）
        │
        ▼  主动结束 / 完成 / 放弃 / 切账号
【周期关闭】→ 进【往期周期】回顾 → 可开新周期
```

## 已敲定的关键决策

- **岔路1 = b**：旅程段**保留对话**，但语境从"分析"切到"执行"；开场分析对话折叠成只读归档，不再回到"重新交简历重新分析"那个开场。
- **岔路2 = b**：进入旅程靠用户**显式点【开始这个学习周期】**，不是计划一生成就自动进入（多一道确认关口，有分界仪式感）。
- **接线改动（认可）**：`generate_plan` 生成计划后**只产出 roadmap 展示、不物化**；用户点【开始】**才**物化成可勾选 Task + 激活 journey。`JourneyState` 需新增 **draft（待确认）** 态（现有只有 active/paused/succeeded/unmet/withdrawn）。
- **岔路3 = a**：把**面经复盘、费曼/出题掌握度**从独立卡片/抽屉**收进执行期对话**，让对话成为旅程的统一陪伴入口（落地愿景的谷底期/碰壁期/掌握期陪伴）。后端判定逻辑（interview / mastery / replan 等 service）**复用不重写**，只是改成对话工具入口。
- **岔路4 = b**：保留**周期级成长回顾**——结束的周期能回看（目标 / 最终匹配度 / 坚持天数 / 完成任务数 / 掌握技能）。这是"往期周期"页，**平移替代**被砍的会话历史，并升维成成长回顾。
- **开场对话归档处理**：① 聊天**原文完整保留、只读、折叠**（给人翻）；② 周期顶部放一张**"本周期设定卡"**，用**现成结构化数据拼**（目标/匹配度/重点缺口/用户偏好/计划周数），既给人快速回忆、也当执行期对话的背景交代；③ **不专门调大模型做"总结"**——干货是现成数据，重跑一遍既多花开销又可能总结错。

## Requirements

**要砍**
- 会话历史模块：历史页、单会话只读回放、侧栏"最近会话列表"、"可反复新建多会话"的概念。
- 独立的**面经卡片 + 掌握度抽屉**的*交互入口*（判定逻辑收进对话；结果仍可在看板/报告呈现）。
- 前端会话持久化的"多会话"模型 → 改为"一周期：一段开场对话（归档）+ 一段执行对话（滚动）"。

**要加**
- 周期状态机 + `JourneyState` 的 draft 态。
- 【开始这个学习周期】确认闸门；物化 + 激活从 `generate_plan` 旁路挪到此动作。
- **第二套对话编排**（执行陪伴期：新系统提示 + 新工具集；工具复用 interview/mastery/replan/tasks/web_search 等现有 service）。
- 开场对话归档（原文只读 + 现成数据拼的"本周期设定卡"）。
- "往期周期"回顾页（替代会话历史）+ 单周期回顾视图。
- "主动结束/完成/放弃周期 → 可开新周期"做实。

**保留不动**
- 开场分析对话 + 三工具 + 两步闸门（原样）。
- 任务看板 / 打卡 / 进度 / 再规划等非对话展示与后端 service。

## 默认决定（已向用户说明，未反对即采用）

- 执行期对话与 journey **一对一**，滚动持久化。
- **本轮不做真账号体系**：单租户现状（设备 id = local）下，"切换账号"暂等价现有机制；只把"主动结束周期 → 开新周期"做实（覆盖绝大多数演示）。
- **可选分期**（待定）：岔路3a 较大，可"面经回灌进对话"先做（最能演示碰壁期陪伴），"费曼/出题掌握度"二期。是否分期在 design.md 阶段定。

## Acceptance Criteria

- [ ] 一个账号同一时间只有一个 active 周期；进入旅程后不再出现"新建会话/回到开场重新分析"的入口。
- [ ] 计划生成后处于"待确认（draft）"，**未**物化 Task / **未**激活 journey；点【开始这个学习周期】后才物化 + 激活，且此前可回改计划。
- [ ] 旅程执行期对话可完成：贴面经→盲区回灌、费曼/出题→掌握判定、"我做不完"→再规划，全部走对话且复用现有后端逻辑。
- [ ] 开场对话进入旅程后变为只读归档，且顶部"本周期设定卡"由现成数据拼出（无额外大模型调用）。
- [ ] 会话历史模块已移除；"往期周期"回顾页可回看已结束周期的成长指标。
- [ ] 前后端测试通过；改动写入 `docs/修改日志/`。
