## 设计要点（route B：可见且可用的闭环）
- **盲区提取**：LLM 优先（`complete_json` 结构化抽「被问倒/不熟的技能 + 严重度」），未配置或
  失败时降级规则（`match_skills` 在面经文本里匹配技能本体）。两路都把技能归一到本体节点；
  与本次分析已识别缺口交叉，命中缺口的盲区升级为 high（面试印证了简历分析里就标出的缺口）。
- **权重回灌**：命中盲区的未完成 Task → `weight` 按严重度提升（high+3/mid+2/low+1，capped 10）
  + `planned_date` 拉到今天，**立即进入「今日任务」**；计划未覆盖的盲区作「建议加练」回包，
  MVP 不自动建任务。
- **匹配口径**：`Task.skill_key` 存的是 roadmap 的 focus 名（非规范 key），故按
  `match_skills(title + skill_key)` 归一后再与盲区 key 比对，提高召回。
- **让回灌可见**（关键）：此前 `weight` 仅 0=至危被消费；本期新增 weight≥2 的消费端——
  前端 `TaskChecklist` 标「🎯 重点」+ 强调样式，配合 planned_date 拉到今天，闭环肉眼可见。

## 审查后修订（对抗式 review 工作流：20 条发现 → 核验确认 13 条，全部修订）
- **LLM 非 dict 崩溃（major）**：模型偶发吐顶层数组/标量时 `complete_json` 调用方 `.get` 抛
  AttributeError → 500。在 `llm.complete_json` 统一加类型守卫（非 dict 即按 LLMUnavailable 降级，
  一处修复所有调用方）；`extract_blind_spots` 再兜一层 `Exception` 记日志后降级规则。
- **回灌静默失败（major）**：路由裸 `except Exception` 无日志 → 线上回灌失败无痕。加
  `logger.exception` 留痕。
- **事务边界（minor）**：`reweight_from_blind_spots` 改为只 `flush` 不 `commit`，由路由把
  「回灌 + matched 回填」单次提交——任一步失败可整体回滚，杜绝「任务已改库、响应却说没回灌」。
- **测试隔离（minor）**：conftest 加会话级 autouse `init_db`，直连 SessionLocal 的用例可独立运行
  （`pytest tests/test_interview.py` 单跑由 3 failed → 8 passed）。
- **跨午夜口径（minor）**：PlanView `todayTasks/overdueTasks` 改为每次重算取当下 `localTodayIso()`，
  与回灌写入的 planned_date 同口径。
- **重点不闪退（minor）**：severity→boost 提到 high+4/mid+3/low+2，保证回灌后被结算降权一次仍 ≥2。
- **陈旧结果（minor）**：InterviewReplayCard 提交前清空上次 result；boosted=0 时文案不再说「提到今天」。
- **严重度同义词（nit）**：medium/critical/严重/轻微… 归一到 high/mid/low。
- **无旅程冗余写库（nit）**：matched 回填移入 journey 分支，无旅程时不再二次 commit。
- **README/测试一致性（nit）**：架构图 8→9 表、测试徽章 83→92；规则路径 severity 断言收紧为恒 mid。

修订后：全量 **92 passed**、单跑 test_interview.py **8 passed**、前端 build 通过、seed 复跑成功。

## 原因
F1 是简历项目里「想再加亮点时单拎」的技术闭环：与 E1 共享 `Task.weight` 靶点，把面试反馈
变成可执行的优先级调整，demo 直观（录完面经，今日任务立刻多出几条 🎯 重点）。

## 抉择
- **route B（提权 + 拉到今天 + 前端高亮）**而非仅提权（闭环不可见）或自动建补强任务（过重、
  涉物化契约）。盲区匹配按 title+skill_key 归一，规避 skill_key 存的是 focus 名而非规范 key。
- **盲区与缺口交叉升级**：复用已有 `skill_gap`，让面试印证的缺口自动升 high，串起分析与复盘。
- **无活跃旅程也存面经**：仅不回灌；回灌失败不影响已存档（鲁棒）。
- **守门子集化**：`interview_logs` 是里程碑三表，不污染 MILESTONE1 基线常量，测试改子集断言。
- **进度接口同口径 today**：createInterview 默认带 localTodayIso()，回灌拉到「今天」与
  E4 修正的 progress/checkins/replan 本地日口径一致。
