## 1. 联网搜索结果可折叠展示

### 修改内容
- 后端 `backend/app/services/agent.py`：`web_search` 工具分支在 `tool_result` 之前新增 `search_results` SSE 事件，结构 `{ id, query, results: [{title, url, snippet}] }`（`id` 关联同一 web_search 工具块，`snippet` 取网页正文前 280 字）。
- 前端契约：`types.ts` 新增 `SearchResultItem`，并把 `PersistedBlock` 的 `tool` 变体扩展出 `query?/results?`；`api/client.ts` 的 `ChatStreamHandlers` 新增 `onSearchResults`，`dispatchFrame` 新增 `case 'search_results'`。
- `ChatView.vue`：`runChat` 新增 `onSearchResults` 回调，按 `id` 用 `findToolById` 把结果挂到对应工具块；工具块模板内渲染**默认折叠**的「搜索结果 N 条」开关，展开后显示每条标题（链接 `target=_blank rel=noopener noreferrer`）+ 摘要；`serializeTurns/deserializeTurns` 往返保留 `query/results`。

### 原因
此前搜索结果仅喂给 LLM、从不发往前端，用户「看不见 AI 搜了什么」。需要把检索透明化，同时不喧宾夺主。

### 抉择
- **复用工具块、以 `id` 关联**，而非新增独立 block 类型：搜索本就属于 `web_search` 工具调用，挂在同一块下语义自然，且最小化持久化模型改动。
- **`resultsOpen` 折叠态为瞬态、不持久化**：重载会话后默认折叠，符合「默认折叠、点击展开」诉求，也避免存盘冗余。
- **后端截断 snippet 到 280 字**：列表展示足够，前端再 `-webkit-line-clamp:3` 双重防溢出。

## 2. 最终主产出 = 完整学习方案（侧栏突出）

### 修改内容
- `ChatView.vue` 侧栏从「仅匹配分析摘要」升级为**双形态成果面板**：
  - 已生成学习方案（`roadmap` 非空）→ 标题切「🎓 你的学习方案」，主体展示「N 周个性化学习路线 + 前 3 周（周号 + 聚焦技能）预览」，匹配度降为底部次级一行；主按钮文案切「查看完整学习方案」。
  - 仅匹配分析（`roadmap` 为空）→ 保留紧凑摘要（环形匹配度 + 目标岗位 + 简述），并新增提示「回答上方问题后将生成完整分周学习方案」；按钮维持「查看完整报告」。
- 新增派生状态：`latestRoadmap / hasPlan / planPreview / fullReportCtaLabel`。

### 原因
用户明确：「匹配报告不是目的，最终输出应是给用户的一份完整学习方案」。原先方案要跳 `/result/:id` 才看得到，在对话流里被边缘化。

### 抉择
- **重点改对话内侧栏，而非重排 `AnalysisReport.vue`（1371 行、被 ResultView/ConversationView 共用）**：侧栏是用户停留的主场景，改动收益最高、风险最低；重排共享报告组件的段落顺序留作后续可选优化，避免一次性动太多稳定组件。
- **CTA 文案随是否已出方案动态切换**：消解「报告 vs 方案」语义混淆。
- **两步流程保持不变**（analyze_match → 反问 → generate_plan）：仅在展示层把方案抬为主角，不动后端编排。

## 3. 对话框组件视觉融合 + 失焦折叠 + 顶栏移除 + 左侧可折叠边栏

### 修改内容
- **左侧边栏**（新增 `components/SideNav.vue` + `shared/appState.ts`，重写 `App.vue`）：
  - 原顶部 sticky 品牌栏整体移除；品牌 Logo、新对话、最近会话列表、历史记录入口移入左侧常驻边栏。
  - 边栏宽屏可折叠成 64px 窄图标栏（`sidebarState.collapsed`，`localStorage` 持久化）；窄屏（≤768px）化为抽屉，由汉堡按钮唤出、点遮罩 / 切路由关闭。
  - 跨组件协同走极简共享单例 `appState`：`newChatSignal`（侧栏新对话 → ChatView 重置）、`conversationsChanged`（ChatView 存盘后 → 侧栏刷新列表）。
- **composer 融合**（`ChatView.vue`）：上传/JD/设置/思考/发送收进单一圆角盒 `.composer__box`；输入框无边框、焦点环落在盒上（`:focus-within`）；主输入行 = 📎附件（带素材角标）+ 输入框 + 发送，**始终一行**；工具行（粘贴 / JD 库 / 设置 / 思考强度）与 chips/展开面板**仅在展开态显示**。
- **失焦折叠**：`composerExpanded = 聚焦 || 某面板开 || 拖拽中 || 已输入文字`；否则折叠为单行。失焦判定用容器 `focusin/focusout + relatedTarget.contains` 避免点内部按钮误折叠。
- 高度模型：去掉 `--header-height` 依赖，`.app-main--chat` 提供 `100dvh`，`.chat` 改 `height:100%`，`.chat__main` 居中限宽 1180px。

### 原因
原 composer 是「附件区 + 工具区 + 输入区」纵向多层堆叠，常驻占位「挤成一坨」；顶栏横亘遮挡对话内容；缺少类 ChatGPT 的左侧导航/会话切换。

### 抉择
- **共享单例 `appState`（reactive + ref）而非引入 Pinia**：仅 3 个跨组件状态，零依赖更轻；信号用自增计数 + `watch`。
- **折叠条件不含「有附件」**：只要有文字/聚焦/开面板才展开，确保「不聚焦时真的只剩一行」；已附素材改用附件按钮上的**角标**提示，兼顾「一行」与「可感知已带素材」。
- **工具行用 `v-show`、展开区用 `v-if`**：工具行频繁切换用 `v-show` 省重渲染；chips/面板较重且含表单，用 `v-if` 避免隐藏态仍占 DOM/触发校验。
- **边栏折叠成窄图标栏而非完全隐藏**：保留 Logo/新对话/历史的常驻可达性；窄屏才用抽屉。
- **新对话用「信号 + 路由」而非直接跨组件调用**：解耦外壳与视图，避免 ChatView 既被路由 `?c=` 又被父组件命令式重置导致的竞态。

## 复核修复（对抗式多维复核 → 据结果修复）

对四维（composer / 外壳侧栏 / 搜索端到端 / 学习方案与回归）并行复核后，修复如下；部分经分析判定为误报/无害者不改。

### 已修
- **[阻断] 双滚动条**（App.vue）：`.app-main--chat` 加 `overflow:hidden`，仅 ChatView 内部 `.chat__scroll` 滚动；窄屏（≤960）放开为页面滚动。
- **[阻断] 搜索结果折叠态初始化**（ChatView）：`deserializeTurns` 与 `onToolCall` 新建工具块时显式 `resultsOpen:false`，杜绝响应式疑虑（重载默认折叠）。
- **[安全] 搜索链接 XSS**（ChatView）：新增 `isSafeUrl()`，仅放行 http/https，危险协议（javascript:/data:）降级为纯文本——搜索结果来自外部 Tavily，视为不可信。
- **[健壮] 后端字段防御**（agent.py）：搜索结果 title/url/content 统一 `str(... or '')`；新增 `_snippet()` 按词边界截断 + 省略号；**空结果不再下发 `search_results` 事件**。
- **[可用性] composer 失焦判定**（ChatView）：`onComposerFocusOut` 改为 `setTimeout` 后以 `document.activeElement` 是否仍在容器内判定，兼容 `<select>`/原生文件框等 `relatedTarget` 为 null 的情形，避免交互途中误折叠；`onFileChange` 选完文件后把焦点交回输入框。
- **[一致性] 工具行渲染**（ChatView）：`v-show` → `v-if`，折叠时真正移出 DOM，与展开区一致。
- **[回归] 学习方案视图保留简述**（ChatView）：`hasPlan` 分支补回匹配分析 `summary`（3 行紧凑截断），避免两步流程下丢失「方案依据」上下文。
- **[竞态] 新对话信号守卫**（ChatView）：`watch(newChatSignal)` 加 `loadingConversation` 守卫。
- **[移动端] 抽屉断点对齐 + 折叠态修正**：App 抽屉断点 768→**960**（与 ChatView 堆叠断点一致，避免「侧栏常驻 + 对话已堆叠」的中间态）；新增 `isWide`（matchMedia 960），SideNav 用 `effectiveCollapsed = collapsed && isWide`，使窄屏抽屉始终展示完整侧栏（而非桌面折叠的 64px 窄栏）。

### 抉择
- **断点对齐选 960 而非 768**：侧栏占 260px，窄屏让侧栏先收成抽屉、把整屏让给纵向堆叠的对话，比「对话拖到 768 才堆叠」体验更顺。
- **`resultsOpen` 仍不进持久化 schema**：Vue 3 Proxy 本就支持动态键响应式，初始化只为消除歧义与可读性；持久化层保持干净。
- **判定为误报/无害未改**：`input.trim()` 触发展开（聚焦时 `composerFocused` 已主导，无闪烁）；`serializeTurns` 写入 `undefined`（`JSON.stringify` 自动丢弃）；工具块 id 透传一致无需 trim。
