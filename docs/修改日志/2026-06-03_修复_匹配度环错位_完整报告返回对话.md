## 1. 修复：匹配度环（ScoreRing）在小尺寸下文字溢出错位

### 修改内容
- `frontend/src/components/ui/ScoreRing.vue`：中心数字 / 「匹配度」文案的字号由**固定值**改为**随环径 `size` 计算**：
  - `valueFontPx = min(30.4, round(size*0.3))`、`captionFontPx = min(12.5, round(size*0.18))`，并以 `:style` 内联绑定；移除 CSS 里写死的 `font-size: 1.9rem / 0.78rem`。
  - 新增 `showCaption = size >= 72`：环太小时隐藏「匹配度」三字，只留中心数字（语义已由 `aria-label` 与相邻文案承载）。
- `frontend/src/views/ChatView.vue`：学习方案侧栏的迷你环改为传 `:stroke="5"`（原默认 12），让 44px 环有更大中心留白，数字稳居其中。

### 原因
`ScoreRing` 的中心数字一直写死 `1.9rem`（≈30px）、副标题 `0.78rem`，与 `size` 无关。大环（120/148px）尚可，但侧栏「学习方案」里的 **44px 迷你环**装不下 30px 数字 + 「匹配度」三字，内容溢出圆环并压到相邻「岗位匹配度 XX%」文案上——即截图中红色圆环与文字错位重叠的 bug。

### 抉择
- **在组件层按 `size` 缩放**而非只在调用处打补丁：迷你环在多处可能复用，根因是组件未自适应尺寸，组件层修复更彻底。
- **用 `min(...)` 封顶**：大环（120/148）算出的字号被夹到与原来完全一致（30.4px / 12.5px），**大环观感像素级不变**，仅小环按比例缩小，零回归风险。
- **小环隐藏副标题**而非强行缩到极小：4~8px 的「匹配度」不可读且更易溢出；隐藏后只保留中心数字最干净，含义由相邻「岗位匹配度 XX%」补足。
- **迷你环显式传细描边 `stroke=5`**：与缩小后的字号匹配，避免 12px 粗环把中心孔挤没。

## 2. 修复：完整报告/学习方案页「返回」应回到原对话而非新建

### 修改内容
- `frontend/src/views/ChatView.vue`：`goToFullReport` 改为 `async`，跳转前若会话尚未落库则先 `await persistConversation()` 拿到会话 id，并以 `query: { from: <会话id> }` 跳转 `/result/:id`。
- `frontend/src/views/ResultView.vue`：读取 `?from=<会话id>`，派生 `backTo` 与 `backLabel`——有来源会话则左上角返回到 `/?c=<会话id>`（续聊态）、文案「← 返回对话」；否则维持 `{ name: 'new' }`、文案「← 返回新建」。错误占位里的返回按钮一并复用 `backTo`。
- `frontend/src/components/AnalysisReport.vue`：新增可选 prop `fromConversationId`；「查看完整报告页」链接据此派生 `fullReportTo`，携带 `?from`。
- `frontend/src/views/ConversationView.vue`：渲染 `<AnalysisReport>` 时传 `:from-conversation-id="Number(id)"`，使会话回看页里的「查看完整报告页」返回也回到该对话。

### 原因
此前 `/result/:id` 的左上角返回写死指向 `{ name: 'new' }`（即 `/`，无 `?c`），而 `/` 无 query 会触发 ChatView 新建空对话——用户从对话点「查看完整学习方案」看完后返回，却丢失了原对话。

### 抉择
- **用 `?from=<会话id>` 显式回传**，而非 `router.back()`：ChatView 在路由跳转时会被卸载，`router.back()` 回到 `/`（无 query）只会重建空对话；必须用 `?c=<id>` 让 ChatView 从后端重新加载该会话，故须把会话 id 带过去。
- **跳转前兜底 `await persistConversation()`**：报告出现时一般当轮已自动保存，但加一道保底确保拿得到会话 id；拿不到（保存失败的极端情形）则回退「返回新建」，不阻断。
- **同步打通 `AnalysisReport`/`ConversationView` 的「查看完整报告页」**：用户原话「或者报告之类的」，统一让所有进入完整报告页的入口都能返回对话，避免只修一处留下不一致。
