## 修改：技能缺口三段改为「上下三行 + 可折叠」

### 修改内容
- `frontend/src/components/AnalysisReport.vue`
  - **布局**：原 `.gap-grid`（3 列等宽网格：必备缺口 / 加分缺口 / 已具备）改为 `.gap-stack`——三段**上下堆叠成三行**的独立面板。
  - **可折叠**：每段头部是整行可点按钮（`gap-section__head`，带 `aria-expanded` + ▾ 旋转箭头），点击切换该段展开/收起；状态用组件内 `reactive({ must, nice, possessed })` 管理。
    - 默认仅展开「必备技能缺口」，「加分缺口 / 已具备」默认收起，以缩短整体高度（用户可按需展开）。
  - **段内卡片**：`.gap-items` 由竖直单列 `flex-column` 改为响应式网格 `repeat(auto-fill, minmax(min(100%,240px),1fr))`——卡片在整行宽度内多列流式排布，窄屏自动回退单列，进一步压缩纵向长度。
  - **语义配色**：保留必备=红 / 已具备=绿 / 加分=默认，移到每段折叠头的背景 + 左侧 3px 色条上。

### 原因
用户反馈：报告页技能缺口部分「太长太难看」，希望「分上下三行布置并且可折叠」。原三列网格在数据多时高度被最长一列撑开、整体冗长，且小屏堆叠后更长。

### 抉择
- **三段折叠面板 + 默认只开必备**：直接命中「三行 + 可折叠」诉求；默认收起加分/已具备，让首屏聚焦最关键的必备缺口，需要时再展开，最大化缩短篇幅。
- **段内卡片改网格多列**而非保持单列：单列在整行全宽下每卡一行会更长；多列流式排布充分利用横向空间，是缩短高度的关键。`minmax(min(100%,240px),1fr)` 兼顾窄屏不溢出。
- **用 `v-show` 而非 `v-if` 收起段落**：缺口数据量有限，`v-show` 切换更顺滑、无重渲染；导出 Markdown（`buildMarkdown`）不受 UI 折叠态影响，仍输出全部内容。
- **未改 `expanded`（报告整体折叠）逻辑**：本次只重排「技能缺口」段内部，报告级「展开详细信息」与完整报告页（`collapsible=false`）行为不变；Result- / ConversationView 复用同组件自动生效。
