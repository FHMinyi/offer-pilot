## 修改：报告/学习方案入口由右下角悬浮角标改为左侧边栏条目

### 修改内容
- `frontend/src/shared/appState.ts`
  - 新增报告导航共享状态 `reportNav = reactive({ available, open, hasPlan })`（ChatView 写入、SideNav 读取）。
  - 新增信号 `openReportSignal` + `requestOpenReport()`：侧栏点击 +1，ChatView watch 到后展开并定位报告面板。
- `frontend/src/views/ChatView.vue`
  - **移除**右下角悬浮角标 `.report-fab`（模板 + 主样式 + 窄屏 `@media(max-width:960px)` 覆盖），及其只被它使用的 `openReportPanel()` 函数（`noUnusedLocals` 会报未使用）。
  - 新增 `watch`：把 `latestReport != null` / `reportPanelOpen` / `hasPlan` 实时同步到 `reportNav`（`immediate`）。
  - 新增 `watch(openReportSignal)` → 调用既有 `focusReportPanel()`（展开面板 + 滚到顶 + 短暂高亮，复用 chip 的定位逻辑）。
  - `onUnmounted` 时 `reportNav.available = false`，离开对话视图后侧栏不残留报告入口。
  - 保留报告面板内的「隐藏」按钮（`hideReportPanel`），隐藏后改由左侧栏入口重新展开。
- `frontend/src/components/SideNav.vue`
  - 在「新对话」与「最近会话」之间新增「报告 / 学习方案」入口按钮：仅在对话页（`onChat`）且 `reportNav.available` 时显示；图标/文案随 `reportNav.hasPlan` 切换（🎓 学习方案 / 📊 匹配分析）；`reportNav.open` 时弱化为「当前所在」态。
  - 折叠态（`effectiveCollapsed`）下收成 40×40 方形图标按钮，与新对话/历史图标对齐。
  - `openReport()` = `requestOpenReport()` + `closeMobileSidebar()`（窄屏点完即收抽屉）。

### 原因
用户反馈：未聚焦输入框时，「报告」按钮与「发送」按钮会贴在一起。根因是报告面板被隐藏后出现的 `report-fab` 用 `position: fixed; right; bottom` 固定在视口右下角，正好与底部 composer 的发送按钮区域重叠（聚焦时 composer 展开变高把发送按钮上推，未聚焦时两者贴近，观感尤为明显）。用户要求「为了美观统一，将报告按钮也放到左侧边」。

### 抉择
- **入口移到左侧栏而非仅调整 fab 位置**：直接命中用户「放到左侧边」「统一」的诉求；左侧栏已是品牌/新对话/历史/会话列表的统一导航区，报告入口归入此处最自然，也彻底消除与发送按钮的重叠。
- **跨组件用既有 `shared/appState` 单例**（不引 Pinia）：与 `newChatSignal`/`conversationsChanged` 同模式，状态由 ChatView 单向写、SideNav 读 + 发信号回传，契约最小。
- **侧栏入口常显（有报告即显示），不只在隐藏时显示**：面板展开时点击=滚动高亮定位，隐藏时点击=重新展开，行为统一，复用 `focusReportPanel()`。
- **保留右侧报告面板与其「隐藏」按钮**：用户只要求改入口位置，未要求动报告面板本身；隐藏→占满宽度→左侧栏重开的闭环自洽。
- **`onChat` 守卫 + `onUnmounted` 双保险**：避免离开对话视图后侧栏残留报告入口；重新进入对话页时 `immediate` watch 重算。
