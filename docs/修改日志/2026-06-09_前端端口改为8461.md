# 修改日志 · 2026-06-09（前端开发端口 5173 → 8461）

> 应需求把前端开发服务器端口从 Vite 默认的 `5173` 改为 `8461`，并同步所有
> 依赖该端口的配置与文档（含后端 CORS 白名单），保证换端口后浏览器不被跨域拦截。

---

## 修改内容

- `frontend/vite.config.ts` — `server.port` 由 `5173` 改为 `8461`，顶部注释同步。
- `backend/app/config.py` — `cors_origins` 默认值 `5173` → `8461`（localhost / 127.0.0.1 两项）。
- `backend/.env.example` — `CORS_ORIGINS` 示例同步为 `8461`。
- `backend/.env`（本地实际生效、未入库）— `CORS_ORIGINS` 三项一并改端口，
  其中保留原有局域网 IP `172.21.160.212`，仅替换端口。
- `scripts/dev-frontend.sh` — 启动提示文案的 URL 同步为 `8461`。
- `README.md` — 快速开始与「打开 …」说明两处 URL 同步为 `8461`。

## 原因

- 用户要求将前端端口统一改为 `8461`。
- 端口不是只在 Vite 一处出现：后端 CORS 白名单按来源 `origin`（含端口）校验，
  若只改前端不改后端，浏览器会在跨域预检时被拦截，接口全部失败。故 CORS 配置必须同改。

## 抉择

- **后端真正读取的是 `backend/.env`（被 gitignore），不是 `.env.example`。** 两者都改：
  `.env` 保证当前本地立即生效，`.env.example` 保证新克隆者拿到正确默认值。
- **保留 `.env` 里的局域网 IP `172.21.160.212` 条目**，仅替换端口——该条目用于远程/局域网
  调试访问（配合 `vite.config.ts` 的 `host: true`），删掉会破坏既有调试链路。
- 未改后端端口（仍 `7968`）与 `VITE_API_TARGET` 代理逻辑——本次只动前端监听端口。
