## 修改：输入框自动聚焦

### 修改内容
- `frontend/src/views/ChatView.vue`
  - 新增 `autoFocusInput()`：`nextTick` 后对 `textareaRef` 调 `focus()`；聚焦会经 composer 的 `focusin` 自动展开输入区。
  - 在三处触发自动聚焦，覆盖「打开即可输入」的各场景：
    1. `onMounted` —— 页面首次打开。
    2. `newConversation()` —— 侧栏「新对话」重置后。
    3. `loadConversation()` 成功分支 —— 切换 / 进入某段会话续聊后。
  - 引入 `onMounted`（vue）与 `isWide`（shared/appState）。

### 原因
用户希望页面 / 新对话打开时光标直接落在输入框,省去先点一下的操作。

### 抉择
- **仅宽屏自动聚焦(`isWide` 守卫)**:移动端自动聚焦会立即弹出软键盘、遮挡内容、打扰浏览,故窄屏跳过(如需全平台聚焦,去掉该守卫即可)。复用已有的 `isWide`(matchMedia 960)单例,不新增状态。
- **统一收敛到 `autoFocusInput()` 一个函数**:挂载 / 新对话 / 续聊三处共用,行为一致、易维护;内部 `nextTick` 保证 DOM 就绪后再聚焦。
- **挂载与 `loadConversation` 可能各聚焦一次(`/?c=<id>` 直接进入时)**:二次 `focus()` 无副作用,换取逻辑简单,不做去重。
- **不动既有的发送 / 上传后回焦逻辑**:那些是操作后的回焦,语义不同,保持原样。
