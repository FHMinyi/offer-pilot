# 求职助手

# 原始想法

> 我现在有一个想法，想做一个服务于在校待实习生的求职助手，能够通过该生目前的简历，和当前的就业环境（这很关键），能够给出就业指导和学习方向，并且和学生进行多轮对话，不仅给出一个学习日程安排表，而且能够进行每天总结，实时调整学习计划，最终拿到offer。而且感觉不仅可以针对在校生，还能针对应届毕业生，跨行业转码者，在职跳槽者等潜在用户

‍

# 核心痛点

如果剥开表象，你这个想法其实精准地击中了在校生求职时的三大核心心理与现实困境：

### 痛点一：极度的“信息迷雾”与“技能错位”（真实且剧烈）

在校生最害怕的不是辛苦，而是“我拼命努力，但方向完全错了”。

学校的培养体系和真实的商业落地之间存在巨大的断层。当一个学生面对全网海量的面经时，往往会陷入迷茫：企业现在到底更看重底层算法的功底（比如能不能手撕动态规划、滑动窗口），还是更看重工程化的实战能力（比如对前端框架底层原理的理解、组件化的设计）？

很多时候，简历投出去石沉大海，正是因为学生的“自我感觉良好”与市场的“真实用人标准”存在错位。

**你的想法价值在于：**  通过将“个人现状”与“当前环境”对撞，打破了这种信息茧房，直接给出了一个明确的“基调”和“靶子”，解决了**不知道往哪打**的痛点。

### 痛点二：“计划破产”带来的连环情绪内耗（这是你想法中最核心的杀手锏）

求职不是一场几天就能结束的短跑，而是一场长达数月的暗夜跋涉。

绝大多数学生在下定决心找实习的第一天，都能制定出一份完美、硬核的复习计划。但现实是，传统的计划表是**静态且脆弱**的。一旦某天因为课业冲突或单纯的状态不好，导致当天的任务（比如计划好的两道算法题没刷，或者某个技术文档没看）没有完成，第二天就会产生巨大的负罪感。这种挫败感积累几天后，往往会导致整个计划的全盘崩溃和彻底摆烂。

**你的想法价值在于：**  “每日总结+实时调整”本质上是在提供一种极高容错率的​**情绪兜底**​。它将庞大且令人窒息的“求职”目标，降维拆解成了每天可以无脑执行的原子级待办事项。今天没做完？没关系，系统会重新计算明天的权重和日程。这极大地降低了执行阻力，解决了**计划易死、执行瘫痪**的痛点。

### 痛点三：暗黑期缺乏“正向反馈”与“陪伴感”

独自准备面试是一个极其孤独且自我怀疑的过程。做完一个个人项目、啃下了一个难懂的知识点，并没有人会立刻给你发Offer，进度条看起来永远是0%。学生极度渴望有人能告诉他：“你今天做得很好，这让你离拿到Offer的概率又提升了”。

**你的想法价值在于：**  多轮对话不仅是为了收集信息，更是充当了一个绝对理性、永远在线的“赛博导师”。它提供了稳定、可预期的正反馈，缓解了​**求职过程中的严重焦虑与孤独感**。

**客观评价：**

你这个想法抓住的根本不是“怎么写一份好简历”这种浅层痛点，你真正抓住的是​**在校生面对庞大且不确定的就业市场时，产生的“失控感”** 。

通过工具，把非标准化的“求职玄学”，转化为像日常打卡流转一样有迹可循的确定性过程，这不仅有实际的痛点，而且痛点极深。

既然痛点是成立的，那么从用户的心理防御机制来看，你觉得当AI某天指出他“进度严重落后，按照当前状态很难拿到Offer”时，这种极度坦诚的反馈是会激发他的斗志，还是会让他直接卸载不用？

# 核心目标用户群体

基于刚才的推演，这款“动态求职与日程规划 AI 助手”的潜在受众可以被清晰地划分为四大类画像。以下是对这四类人群的系统性总结：

#### 1. 在校待实习生（MVP 阶段最佳种子用户）

- **基本标签：**  计算机科班生、前端/后端等领域的开发新人、技术爱好者。
- **行为特征与现状：**  具备一定的技术基础和极客精神，比如自己动手搭过个人服务器、写过 Vue/React 的前端项目，甚至对 AI Agent 的开发和 MCP Server 协议有实操经验。但在向工业界过渡时，不知道如何将这些技术栈与大厂真实的业务需求对齐。
- **核心痛点：**

  - 存在“学术/自嗨项目”与“企业用人标准”的信息差。
  - 面对长线的准备周期，容易陷入“制定完美计划 - 某天没完成 - 全盘放弃”的内耗死循环。
- **产品诉求：**  需要一个类似智能待办（NexusPlan）系统一样的枢纽，把庞杂的基础八股文、算法题拆解成无压力的每日打卡流转，并提供稳定的情绪价值和纠偏。
- **商业定位：**  付费意愿弱但活跃度高，容错率极高，是打磨 AI 对话逻辑和动态日程算法最完美的测试床。

#### 2. 应届毕业生（规模化爆发的核心主力）

- **基本标签：**  面临秋招/春招大考、急需敲定第一份正式工作的高校学生。
- **行为特征与现状：**  时间窗口极度收缩。每天高频刷新各大招聘网站和牛客网，手里捏着多份不同版本的简历。
- **核心痛点：**

  - 投递石沉大海带来的极度焦虑与自我怀疑。
  - 在海投过程中，复习重心容易完全迷失，难以平衡“刷算法题”和“准备特定公司面经”的时间分配。
- **产品诉求：**  高压下的“赛博导师”。要求系统不仅能实时指出自身技能与真实 HC 的匹配度，更要在崩溃边缘提供强有力的任务约束和“兜底式”的进度重建。
- **商业定位：**  具有较高的付费意愿和庞大的基数，痛点最深，是跑通平价订阅制或规模化变现的核心群体。

#### 3. 跨行业转码者（高客单价的重点目标）

- **基本标签：**  非科班出身、试图零基础切入 IT 或其他高薪行业的成年人。
- **行为特征与现状：**  往往已经有一定的工作经验或积蓄，带着极其明确的“改变命运”的目的性入场，执行力通常较强，但对目标行业处于完全的认知黑洞状态。
- **核心痛点：**

  - 完全没有体系化的学习路径，全靠网络碎片拼凑。
  - 极度缺乏专业人士的反馈，根本不知道目前的水平离“拿到 Offer”还有多远。
- **产品诉求：**  保姆级的强力向导。需要工具直接砸下一条明确的生存路线，给出强约束力的每日原子级任务，并且能够事无巨细地解答基础问题。
- **商业定位：**  付费意愿和付费能力双高。为了“确定性的入行结果”，愿意支付高昂的溢价，是未来发掘高客单价定制服务（VIP Coaching）的首选人群。

#### 4. 在职跳槽者（高门槛的终极受众）

- **基本标签：**  具备一定年限工作经验、试图优化职业路径或薪资包的成熟职场人。
- **行为特征与现状：**  备战时间被高强度的日常工作极度切割。不屑于看通用的基础知识，眼光紧盯特定大厂或业务线的精准要求。
- **核心痛点：**

  - 时间成本极其高昂，不允许做无用功。
  - 难以获取精确到“某条业务线具体技术栈和 HC 现状”的深度情报。
- **产品诉求：**  极其敏锐的“雷达”与去水的高效复习板。绝不接受车轱辘话和无效的冗余规划，AI 必须能针对高阶工程化难题或系统设计给出极其刁钻的面试点拨。
- **商业定位：**  钱包最厚，但对数据精准度和 AI 专业度的要求极高。适合作为产品大成后，打通高端垂直数据源才去触碰的终极受众。

# 完整的用户场景

### 产品愿景与核心灵魂

本产品旨在为寻求实习机会的在校生打造一款“动态求职管家”​ **。 表面上，它解决的是“简历匹配”和“日程规划”的效率问题；但在底层，其真正的产品灵魂是**“情绪管理与动态纠偏”。它将非标准化的、充满玄学的求职过程，转化为类似任务枢纽（如枢纽待办系统）一样有迹可循、高容错率的确定性流转。

### 一、 核心用户旅程与情绪触点映射

求职不是一天的冲刺，而是一场长达数月的马拉松。产品需要精准捕捉用户在五个核心阶段的心理变化，并提供相应的系统干预策略：

|**求职阶段**|**用户真实处境与心理状态**|**产品的核心介入场景**|**关键情绪触点 (情感转化)**|
| ------| ------------------------------------------------------------------------------------------------------------------------------------| ------------------------------------------------------------------------------------------| ------------------------------------------------------|
|**1. 触发期**<br /><br /> *(盲人摸象)*|看着周围人投简历感到发虚；不清楚学校学的基础知识（如基础JavaScript）与企业真实需求（如 Vue 3、TypeScript、工程化实战）之间的差距。|简历与市场JD对比，量化能力缺口，生成第一版结构化的技能填坑路线图。|**恐慌 ➔ 掌控感**<br /><br />将未知的恐惧，转化为量化的差距和明确的行动靶子。|
|**2. 蜜月期**<br /><br /> *(充满干劲)*|刚立下目标，多巴胺分泌旺盛，愿意投入时间执行计划。|每日派发原子级任务（如：刷2道滑动窗口算法题，复习 Vue 响应式原理），完成即给予正向激励。|**目标感 ➔ 成就感**<br /><br />用极短反馈链路的每日打卡，提供稳定的情绪价值。|
|**3. 谷底期**<br /><br /> *(计划破产)*|突发课业冲突，或在动态规划等难点上卡壳，连续几天未完成计划；产生严重的内疚感与逃避心理。|察觉中断后，不发警告，而是主动提供心理疏导，并​**自动降权、重组接下来的任务**。|**内疚/逃避 ➔ 被接纳与振作**<br /><br /> *(核心护城河)* 提供“情绪兜底”，接住崩溃，降低重启成本。|
|**4. 碰壁期**<br /><br /> *(实战毒打)*|开始面试，因紧张或遇到知识盲区被挂；深陷自我怀疑。|引导用户复盘面经，AI 提取盲区，将“失败”定义为数据收集，并据此调整后续学习权重。|**自我怀疑 ➔ 恢复理性**<br /><br />将情绪化的挫败，转化为客观可执行的工程任务。|
|**5. 终局期**<br /><br /> *(拿到Offer)*|历经数月，终于签下实习合同。|极具仪式感的庆祝，回顾数月来的代码行数、刷题量和坚持打卡的天数。|**激动 ➔ 深度认同**<br /><br />为一段漫长且痛苦的旅程画上圆满的句号。|

### 二、 关键产品策略：动态人设引擎 (Persona Engine)

为了在上述不同的旅程阶段精准击中情绪触点，系统将采用“全都要，让用户自己选”的 AI 助手人设策略。这不仅赋予了用户极大的掌控感，还能在工程架构上实现底层的“理智脑”（任务调度）与前端的“情感脑”（交互外壳）的完美解耦。

#### 1. 人设矩阵与场景匹配

- **严厉的教练（适合“蜜月期”与“冲刺期”）：**

  - **交互风格：**  强调纪律、压迫感和目标导向。
  - **应用场景：**  当用户需要极强的执行力去攻克底层算法或硬核知识时。打卡失败会给出直击痛点的后果分析（例如：“今天算法题未完成，意味着你在大厂核心业务线过筛的概率下降了”）。
- **温柔的学长/学姐（适合“谷底期”）：**

  - **交互风格：**  强同理心、经验分享、情绪抚慰。
  - **应用场景：**  当用户连续中断计划、产生内耗时。通过分享“过来人的惨痛经历”降低防御，引导用户从最简单的任务重新开始。
- **理性的系统管家（适合“碰壁期”）：**

  - **交互风格：**  客观、数据驱动、冷静分析。
  - **应用场景：**  面试失败后的复盘。剥离情绪，直接从面经中抓取技术盲点（如特定的网络协议细节），重构第二天的日程表。

#### 2. 高阶演进：动态自适应切换

在 MVP（最小可行性产品）阶段，由用户在设置中手动切换人设。在后续迭代中，系统可通过多轮对话的语义分析或任务积压的阈值触发，**主动建议或平滑过渡**人设（例如：察觉到用户极度焦虑时，自动从“教练”切换为“学姐”进行安抚），从而实现真正意义上的“求职陪伴”。
