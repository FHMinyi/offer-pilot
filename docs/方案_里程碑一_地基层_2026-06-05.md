## 1. 方案总览

**选定策略**：走路骨架 · 单用户有状态闭环（Thin Slice），并嫁接渐进方案的「稳定主键物化契约」、Full Foundation 的「单点防越权形 / SSE user_id 显式穿透 / structured 双路径复用 / Task 四态枚举」。评委双双推荐（总分 33 / 34，第二名渐进方案 27，Full Foundation 19）。

**一句话定位**：用最低风险、最快地点亮「Task 勾选 → 每日打卡 → Progress/Streak 回灌」这条**有状态闭环**并可端到端演示；账号只焊 `user_id` 字符串接缝、**暂不通电**，地基正确性提前锁定，里程碑三接真鉴权只换实现不动表结构。

**本期目标（In Scope）**
- 新增 3 张有状态表：`JourneyState`（旅程主表）/ `Task`（roadmap 物化的可勾选行）/ `CheckIn`（每日打卡）。全部 day-one 带 `user_id String(64)`，与未来真实 id 同形。
- `get_current_user`（X-Device-Id → 'local'）零鉴权归属接缝 + `require_owned`/`scope_to_user` 单点防越权「形」（本期放行）。
- roadmap 裸字符串 → Task **幂等物化契约**（稳定主键 + merge-upsert + skipped 软删），治本期最大风险。
- 只读聚合 `GET /progress`（实时聚合 Task + 惰性算 streak，无 scheduler）+ 读写端点。
- SSE `user_id` 显式穿透 `chat → run_turn → _persist_run → materialize`（本期值恒为 'local'，链路焊死）。
- `Resume.structured` 复用接缝，覆盖**引用模式 + chat 两条路径**，杜绝每次从 raw_text 重解析。
- 前端：`X-Device-Id` 单点注入 + Task 勾选 / 每日打卡 / 进度看板 UI。
- `backend/migrations/` 目录 + `db_guard.verify_schema()` schema 守门（Alembic 的零成本过渡设施）。

**本期非目标（Out of Scope）**
- 真实账号体系（注册/登录/密码/JWT/OAuth/session）。
- 数据隔离强制鉴权：`scope_to_user`/`require_owned` 本期**放行**；`conversations.py:45`、`analysis.py:78` 全表返回越权点**保留**并标注 `KNOWN-MULTI-TENANT-LEAK (milestone 3)`。
- 动态再规划引擎、人设引擎（里程碑二）；面经回灌、真实环境感知、四类画像（里程碑三）。
- Alembic、scheduler/cron、给旧 5 表加 `user_id` 列、并发乐观锁。

## 2. 核心抉择（决策 / 选择 / 理由）

| 决策 | 选择 | 理由 |
|---|---|---|
| **是否现在做完整账号** | **否**。只焊 `user_id String(64)` 接缝，`get_current_user` 从 `X-Device-Id` 取归属标签，缺省 `'local'`，零鉴权 | 付费意愿未验证的学生测试床，本期该验证「有状态闭环用户买不买账」，而非账号建得对不对。String 与未来真实 id 同形，里程碑三只换 `get_current_user` 实现 + 一次性 `UPDATE 'local'→真实id`，**不改表结构/路由签名**。Full Foundation 强登录会让 41 个无 header 测试几乎全红、破坏「打开即用」 |
| **是否引入 Alembic** | **否**。改为建 `backend/migrations/`（幂等手写 SQL + README，本期仅放基线蓝本不执行）+ `db_guard.verify_schema()` 守门 | 本期全是**纯新增表**，`init_db()` 的 `create_all` 零摩擦（已核实 `database.py:42` 只建缺失表、绝不 ALTER 旧表），连一行 ALTER 都不需要。Alembic 唯一刚需（给已存在表加 NOT NULL 外键 / batch_alter_table）本期不存在，引入它反与 `conftest.py` 的 lifespan+create_all 建表路径冲突、把 SQLite↔PG 双环境风险提前。Alembic 连同旧表回填一并推迟到里程碑三 |
| **AnalysisRun 与活计划如何衔接** | **锚定 AnalysisRun，不新建 Plan 主表**。`Task.analysis_run_id` 必填 FK；`AnalysisRun.result` 保留作报告卡渲染源，Task 是其可勾选投影 | AnalysisRun 已承载 resume_id/job_ids/target_role/weeks/result(含 roadmap)；`run.id` 已通过 SSE `report` 事件（`agent.py:418`）与 `analysis.py` 透传到前端（嫁接渐进方案最锋利洞察）。避免 roadmap 双写主表、零回归报告卡 |
| **roadmap→Task 物化对账** | 业务键 `(analysis_run_id, week, order_index)` + `UniqueConstraint`；重跑 generate_plan 走 **merge-upsert 保留 status/done_at**，周次缩减把多余旧 Task 标 `status='skipped'`（软删不物删） | 治本期 **biggest_risk**：演示中最易触发的「调周数/换 JD 重跑」路径。roadmap.py 是确定性纯函数、同输入列表顺序稳定，下标对账可在措辞微调时保留进度；软删保证 `CheckIn.completed_task_ids` 引用永不悬空 |
| **Task 完成态字段** | **四态枚举 `status: todo/doing/done/skipped` + `done_at` + `order_index` + `weight`**，不设 `done` 布尔（完成判定统一 `status=='done'`） | 嫁接 Full Foundation：`skipped/doing` 是里程碑二纠偏算法必需输入信号，`order_index`/`weight` 是重排/降权写入靶点。比布尔几乎零成本却避免里程碑二再加列。前端 TS 可带冗余 `done` 便利字段（=`status==='done'`，后端一并返回） |
| **CheckIn 引用任务方式** | `completed_task_ids: JSON(list[int])` 引用 **Task.id 稳定主键**，绝不用 week+index/title | 嫁接稳定主键：roadmap 措辞变化或重排后打卡仍可靠回指，进度不丢。当日快照整体读写、单用户量小，用 JSON 而非中间关联表 |
| **Progress/Streak 是否建表** | **不建表**。`GET /api/progress` 实时聚合 Task + **惰性按 CheckIn 日期连续性算 streak** | 对齐硬约束「`main.py` lifespan 仅 init_db、无 cron」。单用户聚合成本可忽略，预留 `ProgressSnapshot` 命名供后续缓存 |
| **是否给旧 5 表加 user_id** | **否**，旧表结构一字不动 | `create_all` 不会改旧表，加列须手写 ALTER（本期唯一「不可逆」动作），且回填有丢数风险、牵动 41 个无 header 测试。接缝正确性由「新表 day-one 带 user_id」保证，旧表归属推迟到里程碑三一次性回填 |
| **SSE user_id 处理** | 路由层 `Depends(get_current_user)` 解出，作**显式参数**穿透 `run_turn(..., user_id=) → _dispatch_tool → _persist_run / materialize`，全带默认 `'local'` | `chat.py:34` 在生成器内自建 `SessionLocal()` 脱离请求依赖；若里程碑三才补穿透，极易写无归属脏行。本期值恒 'local' 但链路 + 测试一次焊死，几乎零成本 |
| **structured 复用范围** | `pipeline` 加可选 `resume_structured`；`analysis.run_analysis`（引用模式）+ `agent._persist_run`（chat 路径）两处都注入 | 嫁接 Full Foundation「复用覆盖引用 + chat 两路径」，避免渐进方案自承的「只补一处闭环不完整」，省 LLM 重解析成本，为里程碑二增量更新铺路 |

## 3. 数据模型

### 3.1 关系草图

```
user_id (String(64) 归属标签，本期= 'local'/device-id，里程碑三= 真实 users.id)
  └─ JourneyState (单用户取最新 active 一行)
       ├─ Task (N)  ←物化自→ AnalysisRun.result.roadmap (旧 JSON 保留, 兼容报告卡)
       │     └─ (溯源) analysis_run_id → AnalysisRun (现有表, 不加列/不加 FK 约束)
       └─ CheckIn (N, 每日一行, UNIQUE(user_id,date))
              └─ completed_task_ids JSON[int] → 引用 Task.id (稳定主键)
  Progress = Task.status 聚合 + CheckIn 序列惰性算 streak (不建表)
```

现有 5 表（`Resume / JobPosting / AnalysisRun / Conversation / SavedJd`）**零改动**。

### 3.2 Mapped 声明式（追加到 `backend/app/models.py`，不动现有 5 类）

顶部补：`from datetime import date`；`from sqlalchemy import Date, UniqueConstraint`（`Integer/String/Text/JSON/DateTime/ForeignKey` 已在用）。

```python
class JourneyState(Base):
    """旅程主表：单用户取最新 active 一行（约定，非约束）。"""
    __tablename__ = "journey_states"
    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    user_id: Mapped[str] = mapped_column(String(64), index=True, default="local")
    analysis_run_id: Mapped[int | None] = mapped_column(ForeignKey("analysis_runs.id"), nullable=True, index=True)
    target_role: Mapped[str] = mapped_column(String(255), default="")
    stage: Mapped[str] = mapped_column(String(32), default="executing")   # 五阶段(C2): diagnosing/executing/applying/interviewing/closing
    status: Mapped[str] = mapped_column(String(16), default="active")     # active/archived
    start_date: Mapped[date | None] = mapped_column(Date, nullable=True)  # 自然日(C4)，streak/planned_date 起算
    planned_weeks: Mapped[int] = mapped_column(Integer, default=4)
    current_week: Mapped[int] = mapped_column(Integer, default=1)         # 里程碑二靶点
    last_replanned_at: Mapped[datetime | None] = mapped_column(DateTime(timezone=True), nullable=True)  # 里程碑二靶点
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), default=_utcnow)
    updated_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), default=_utcnow, onupdate=_utcnow)
    tasks: Mapped[list["Task"]] = relationship("Task", back_populates="journey", cascade="all, delete-orphan")


class Task(Base):
    """roadmap 物化为可勾选行——闭环核心 + 物化契约锚点。"""
    __tablename__ = "tasks"
    __table_args__ = (UniqueConstraint("analysis_run_id", "week", "order_index", name="uq_task_run_week_order"),)  # C3
    id: Mapped[int] = mapped_column(Integer, primary_key=True)            # 稳定主键：CheckIn 引用它
    user_id: Mapped[str] = mapped_column(String(64), index=True, default="local")
    journey_id: Mapped[int | None] = mapped_column(ForeignKey("journey_states.id"), nullable=True, index=True)
    analysis_run_id: Mapped[int] = mapped_column(ForeignKey("analysis_runs.id"), nullable=False, index=True)
    week: Mapped[int] = mapped_column(Integer, default=1)
    order_index: Mapped[int] = mapped_column(Integer, default=0)         # 该 run 该 week 内跨 kind 连续序(C3)
    skill_key: Mapped[str] = mapped_column(String(64), default="")
    title: Mapped[str] = mapped_column(Text, nullable=False)             # 原 roadmap 裸字符串
    kind: Mapped[str] = mapped_column(String(16), default="learn")       # learn/deliverable/interview/review
    weight: Mapped[int] = mapped_column(Integer, default=1)              # 里程碑二重排/降权靶点
    status: Mapped[str] = mapped_column(String(16), default="todo")      # 四态 todo/doing/done/skipped
    planned_date: Mapped[date | None] = mapped_column(Date, nullable=True)
    done_at: Mapped[datetime | None] = mapped_column(DateTime(timezone=True), nullable=True)
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), default=_utcnow)
    updated_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), default=_utcnow, onupdate=_utcnow)
    journey: Mapped["JourneyState | None"] = relationship("JourneyState", back_populates="tasks")


class CheckIn(Base):
    """每日打卡：同 user_id+date 唯一，upsert；引用 Task.id 稳定主键。"""
    __tablename__ = "check_ins"
    __table_args__ = (UniqueConstraint("user_id", "date", name="uq_checkin_user_date"),)
    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    user_id: Mapped[str] = mapped_column(String(64), index=True, default="local")
    journey_id: Mapped[int | None] = mapped_column(ForeignKey("journey_states.id"), nullable=True, index=True)
    date: Mapped[date] = mapped_column(Date, index=True, nullable=False)  # 客户端本地自然日
    mood: Mapped[str] = mapped_column(String(16), default="")
    note: Mapped[str] = mapped_column(Text, default="")
    minutes: Mapped[int] = mapped_column(Integer, default=0)
    completed_task_ids: Mapped[list] = mapped_column(JSON, default=list)  # list[int]，引用 Task.id（非 week/index）
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), default=_utcnow)
    updated_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), default=_utcnow, onupdate=_utcnow)
```

> `Progress` 不建表。`GET /api/progress` 实时聚合 Task（按 status/week/stage）+ 由 CheckIn 序列惰性算 `current_streak/longest_streak`。已预留 `ProgressSnapshot` 命名供后续缓存。

### 3.3 等价 DDL（SQLite，PG 无专有特性）

`create_all` 自动生成；`order_index` 已避开 SQL 保留字（不再用 `order`）。三表均：自增主键、`user_id` 默认 `'local'` 带索引、外键 `analysis_run_id→analysis_runs(id)`、`journey_id→journey_states(id)`；`tasks` 唯一约束 `(analysis_run_id, week, order_index)`、`check_ins` 唯一约束 `(user_id, date)`；为 `user_id/analysis_run_id/journey_id/date` 建索引。**`Date` 在 SQLite 上存为文本，前端传 `YYYY-MM-DD` 本地日入库。**

### 4.1 共享接缝（新文件）

**`backend/app/deps.py`**
```python
from fastapi import Header
DEFAULT_USER = "local"

def get_current_user(x_device_id: str | None = Header(None, alias="X-Device-Id")) -> str:
    """里程碑一：归属标签，非鉴权。里程碑三换成 token→users 表→真实 user_id，签名不变。"""
    uid = (x_device_id or "").strip()
    return uid[:64] if uid else DEFAULT_USER
```

**`backend/app/ownership.py`**（单点防越权「形」，本期放行）
```python
from fastapi import HTTPException

def scope_to_user(stmt, model, user_id):
    # 里程碑一：放行（不加 where）。里程碑三：return stmt.where(model.user_id == user_id)
    return stmt

def require_owned(db, model, obj_id, user_id):
    obj = db.get(model, obj_id)
    if obj is None:
        raise HTTPException(status_code=404, detail=f"{model.__name__} 不存在。")
    # 里程碑三：if getattr(obj, "user_id", None) != user_id: raise HTTPException(403, "无权访问。")
    return obj
```

```python
    total_tasks: int; done_tasks: int; completion_rate: float   # 0~1
    current_week: int
    week_progress: list[WeekProgressItem]
    current_streak: int; longest_streak: int
    last_checkin_date: date | None = None
    checked_in_today: bool
```

> 另给现有 `AnalysisRunRequest` **追加** `prefer_structured: bool = Field(True, description="引用模式优先复用 Resume.structured，缺失才回退重解析")`（可选默认 True，现有 `test_api.py` 不传不受影响）。

**streak 惰性算法**（无 scheduler）：取该 user `CheckIn.date` 升序去重列表 `dates`；`current_streak` 从今天（或昨天）往回数连续日，`last==today` 或 `today-1` 才计否则 0；`longest_streak` 扫一遍求最长连续段；`checked_in_today = today in dates`；`current_week = clamp(1, (today - start_date).days // 7 + 1, planned_weeks)`。
