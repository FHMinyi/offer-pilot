# OfferPilot 差距分析 · 对照原始愿景（2026-06-05）

> 方法：把 `docs/OfferPilot_原始想法.md` 的愿景拆成 **12 个能力维度**，每个维度先在代码库实证映射、再由另一名 agent 独立**对抗式复核**（挑战高估与低估两个方向），最后加权汇总。
> 共 34 个子 agent、~120 万 token、~120 次代码查证。结论忠于代码证据。

---

## 一句话结论

**现在是一个「一次性简历↔JD 匹配诊断 + 静态周计划生成器」**——单机、无登录、离线可跑、工程扎实的工具；
**愿景要的是「数月级、有情感脑、随状态实时重算的动态求职陪伴管家」**。
产品做出了愿景的**骨架（触发期诊断）**，却几乎完全没有它的**灵魂（情绪管理与动态纠偏）**。

**整体达成度 ≈ 22%**（按“是否触达愿景灵魂”加权；若只数“有没有雏形”，12 个维度里 5 个有、7 个缺）。
**灵魂达成度 ≈ 3–5%，近乎为零。**

## 灵魂评估：情绪管理 + 动态纠偏 ≈ 3–5%

愿景在原文里把灵魂明确定义为「情绪管理与动态纠偏」，对照代码逐项核实——

1. **动态纠偏完全缺失**：`roadmap.generate()`（roadmap.py:15-56）是一次性按优先级切周的纯函数，task 是裸字符串、无 `done/weight/完成日` 字段；`generate_plan` 生成后整盘写回 `AnalysisRun.result` 即终止。全后端 grep 不到 `recompute/replan/降权/重组/snapshot`——**“今天没做完→重算明天权重”这条护城河逻辑根本不存在**。
2. **情绪管理无载体**：models.py 仅 5 表，无 `Mood/Stage/CheckIn/DailyLog/Progress`。没有“状态”概念就谈不上情绪干预；权重 `_priority/_score`（gap_analysis.py）只由 JD 频率与简历技能决定，**无任何来自面试/情绪/进度的回灌入参**。
3. **人设引擎为零**：agent.py:145-169 是唯一且硬编码的系统提示词，受众写死“应届生/实习”、风格写死“中文简洁”，无人设矩阵、无切换入口（`ChatContextIn` 无 persona 字段）、前端零相关。唯一旋钮 `reasoning_effort` 经核实只是“思考算力预算”，**不是人格**。

> 现有所有“复盘/模拟面试”字样均为**计划内自检周**（roadmap.py:111-127），由“简历 vs JD 缺口”驱动，**与真实面试结果零关联**——不是愿景说的碰壁期复盘。

## 用户旅程五阶段覆盖度

| 阶段 | 覆盖度 | 说明 |
|------|:----:|------|
| **1 触发期**（盲人摸象→掌控感） | **~70%** | 覆盖最好。量化缺口+匹配度+明确靶子端到端落地。折扣：靶子所对的“市场 JD”全靠用户手动粘贴（非真实环境感知）；缺口只二值，无“当前熟练度 vs 目标”梯度 |
| **2 蜜月期**（干劲→成就感） | **~10%** | 能出按周静态计划，但**无每日原子任务、无打卡、无完成态、无即时激励**。只有“一张计划表”，没有“每天可执行+即时反馈”的循环 |
| **3 谷底期**（计划破产→被接纳） | **~2%** | **核心护城河，几乎为零**。无中断察觉、无降权重组、无温柔学长学姐人设。对最深痛点“计划易死→摆烂”零作为 |
| **4 碰壁期**（实战毒打→恢复理性） | **~3%** | 实质不存在。无任何承接真实面经/面试结果的入口、模型、工具、API；权重无面试信号回灌 |
| **5 终局期**（拿到 Offer→深度认同） | **~0%** | 完全缺席。无庆祝、无“数月代码行数/刷题量/打卡天数”回顾（这些累计指标的数据模型根本不存在） |

## 已扎实的地基（可放心复用）

- **简历↔JD 匹配与技能缺口量化**是全库最强、实跑非 stub：`gap_analysis.analyze()` 产出必备/加分/已具备三桶，对齐愿景三分。
- **可解释的匹配度公式**真实落地：必备 0.8 + 加分 0.2；项目/实习支撑算强证据 1.0、仅技能罗列算弱证据 0.5；缺口带 priority/reason/required_by/frequency。
- **全链路 LLM 优先 + 规则降级、离线可跑**：无 Key 也能端到端出报告，工程鲁棒性好。
- **对话 Agent 编排可控、流式体验完整**：3 工具分两步、SSE 实时反馈、会话可持久化续聊。
- **技能本体**有 ASCII 边界正则防误匹配，且新兴技能 free-form 不丢弃——是日后接“动态市场本体”的可用起点。
- **联网搜索雏形已打通**（Tavily）并优雅降级——是“环境感知”维度未来强化的接入点。

## 务实路线图：从“快照工具”到“动态管家”

### 里程碑一：从「快照工具」变「有记忆的旅程」（地基层，硬前置）
> 不做这层，后续全部能力都无处落脚。

- 新增 **User 表** + 为 5 张业务表补 `user_id` 外键；`conversations` 列表查询改为按 user 过滤（现 conversations.py:45 全表返回）；接最简注册/登录 + token 中间件
- 新增 **Task 表**（原子任务 + 状态 todo/done/skipped + weight + 计划日/实际完成日 + 所属 week）——把 roadmap 的裸字符串 task 升级为可持久、可勾选、带权重的实体
- 新增 **JourneyState/Stage**（触发/蜜月/谷底/碰壁/终局 + 转移条件）与 **Progress/CheckIn/DailyLog**（打卡 + streak + 累计指标）
- 打通 `structured` 复用闭环：pipeline 改为读 `Resume.structured` 而非每次从 `raw_text` 重解析；前端可见可编辑结构化简历

**解锁**：蜜月期“每日任务+打卡+完成态”与终局期“数据回顾”的数据底座；触发期诊断进入用户长期档案。

### 里程碑二：注入灵魂——动态纠偏 + 情感脑（护城河层）
- **动态再规划引擎**：每次打卡/每日结算后基于 Task 完成情况**增量重算**剩余周权重与日程（降权未完成、顺延、重组），而非现在每次从零生成——兑现核心护城河
- **旅程状态机**：按任务积压阈值/中断天数/对话语义判定阶段并触发干预（谷底期主动疏导 + 降权重组，不发警告）
- **动态人设引擎 MVP**：3 套人格（严厉教练/温柔学长学姐/理性管家）提示词模板 + `ChatContextIn` 加 persona 入口 + 前端人设选择器（拆理智脑/情感脑两层）
- **正反馈可视化**：真实进度条/坚持天数/离 Offer 概率，替换现仅静态匹配度的 ScoreRing

**解锁**：谷底期“情绪兜底+自动降权重组”（核心护城河）、蜜月期“即时正向激励”、全程“陪伴感”——产品第一次拥有灵魂。

### 里程碑三：闭合碰壁期 + 接真实环境 + 画像分层（差异化层）
- **面经复盘闭环**：新增面试/盲区表 + 提交面经的端点；LLM 从面经抽取盲点映射到 skills key；`gap_analysis._priority/_score` 接受“面试暴露弱点”作为额外加权；据此增量重排剩余周
- **真实环境感知**：按目标岗位+地域+时间窗自动抓取/聚合真实在招 JD（启用 source_url 抓取）；让 search 直接驱动分析引擎（替换依赖 LLM 自觉填 extra_jd 的脆弱路径）；新增 market/HC/薪资/需求频次模型 + 定时刷新（引入当前完全缺失的 scheduler 层）
- **四类画像分层**：识别在校/应届/转码/在职，差异化产出强度与语气；自动建议人设切换
- **商业化骨架**：平价订阅 vs VIP Coaching 的 plan/tier 权益模型

**解锁**：碰壁期完整闭环、触发期“靶子”升级为真实 HC 对撞（打破信息茧房）、四类用户差异化陪伴与可变现。

---

## 一句话回答“还有多远”

**功能骨架已搭好约 1/4，但产品的“灵魂”尚未动工。**
你已经把最难的“量化诊断”做扎实了；真正的护城河（数月级动态纠偏 + 情绪陪伴 + 人设）几乎是从零开始，且都卡在**同一块缺失的地基——“长期有状态的数据模型 + 账号归属”**上。先补这块地基，灵魂才能落地。
