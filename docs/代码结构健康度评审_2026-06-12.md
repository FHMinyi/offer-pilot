## 三、高危发现（仅 1 条，3 个维度独立核实）

### H1. `ChatView.vue` 是教科书式 god component ✅✅✅

- **文件**：`frontend/src/views/ChatView.vue`（3864 行 = script 1360 + template 794 + scoped style 1708，样式占全文件 44%）
- **证据**：占整个前端代码（12151 行）的 **32%**，是次大文件 AnalysisReport.vue（1464 行）的 2.6 倍、后端最大文件 agent.py（610 行）的 6 倍。script 内约 69 个函数、清点出约 13 项职责：有序 blocks 渲染模型、SSE 流回调（runChat 约 152 行）、会话序列化/反序列化/自动保存/续聊（约 230 行）、智能滚动、语气滑块+localStorage、附件/JD 库管理、BYO LLM 设置面板、PDF 上传+拖拽、composer 聚焦折叠、token 用量累计、报告侧栏与 SideNav 信号协同、编辑/重试/停止。核实者另证实：它是近期 E3 / BYO LLM / usage 三个特性的共同落点——本次工作区「用量小字」功能在此文件产生 14 个 hunk（+87 行）横跨三段，**改动汇聚效应仍在加剧**。
- **缓解因素**：作者的分节注释（`===` 分隔线分 6 大节）与 SSE 回调只改响应式对象的解耦设计，使拆分难度仅为**中等**。
- **建议**（两步走，可渐进，不必一次重构）：
  1. 先抽 composables：`useConversationPersistence`（serialize/deserialize/persist/load 约 230 行几乎零 DOM 依赖，最佳第一刀）、`useChatStream`、`useSmartScroll`、`useResumeUpload`；
  2. 再抽子组件：`AssistantBlocks`（见 M1，一并解决双份渲染）、`ChatComposer`、`ReportSidePanel`，各自带走对应 scoped CSS。目标压到 1000 行以内。
